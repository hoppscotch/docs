Documenting your API allows collaborators to get a better understanding of the functionality of your APIs and helps support developer adoption of your public APIs.

You can automatically generate documentation for your API collections, then share the documentation with [GitHub Gists](https://docs.github.com/en/github/writing-on-github/editing-and-sharing-content-with-gists/creating-gists).

<alert>This feature requires you to log in with your GitHub account.</alert>

### Documenting an existing collection

<img src="/documentation/document-dark.png" class="dark-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>
<img src="/documentation/document-light.png" class="light-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>

To generate and view documentation for local collections, click **Import** and select the `.JSON` file that contains your collection.

### Documenting a collection

<img src="/documentation/fromcollection-dark.png" class="dark-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>
<img src="/documentation/fromcollection-light.png" class="light-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>

Select the collection you want to document, then click **Generate documentation**.

You can then see a preview of your documentation.

### Creating a gist

<img src="/documentation/local-dark.png" class="dark-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>
<img src="/documentation/local-light.png" class="light-img" height="1280" width="640" alt="API DOCUMENTATION IMAGES"/>

Click **Create secret gist** at the top of the documentation preview. This will create a well structured secret gist for your API documentation.

**Note:** Secret gists aren't private, so can send the URL of a secret gist to your collaborators. However, secret gists do not show up in GitHub's Discover gists and are not searchable.
